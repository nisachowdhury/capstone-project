ANALYSIS: This case focuses on testing two different approaches in a accessibility tool case study. When it comes to accessibility tools and issues that arise, designers have to determine which tools to build and develop. Using data collection, how can we determine which of these two tools is better to use? In many cases, developers utilize the design cycle, where at testing stages implement and test the usability and efficiency of tools, run uer tests, go back and make changes, and continue.

HOW TO LOOK AT THIS DATA: Here we see the number of seconds it takes users to complete the finding game in both cases. To understand which method works better, we need to think which allows the user to complete the task in the least amount of time. One of the main goals of accessibility is to allow users to save time and energy in the elements they need access to make things easier. In this case, looking at both, we need to compare which dataset has the higher count of the smaller amount of seconds. One thing to consider is, just because a tool saves us time, does this mean it is easier to use?
